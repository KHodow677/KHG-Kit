Introduction

Isle of Lore 2: Terrain Icons is a pack of 26 icons and mini icons in various shapes. They can be used as chapter headers, decorative elements, interface icons or in your map's legend, or as in-text icons in case you want to reference terrains or biomes, like the Pine Forest from my Isle of Lore 2: Hex Tiles pack!

The icons come as png files and also as Krita (.kra) and Photoshop (.psd) source files. Both the icons and the mini icons are available in square, circle and diamond shapes. The big icons additionally come with a thick or a thin border and also with or without a name overlayed. The pack is compatible with my batch image editing tool Color Mapper, which means you can easily change the color palette of the icons to your own liking... the required Color Mapper Files are included!

If you bought the pack when it came out, you may notice some differences. First and foremost, the color palette is a bit darker and more saturated, making it compatible with the Isle of Lore 2: Hex Tiles update. The icons were completely re-drawn in a higher resolution (200% the size), too! Besides that, some new icons (Oak Forest, Mixed Forest, Meadow, Jungle, Valley and Wheat) were added. Some of the superfluous border variants (dark thick, colored thick), name variants (smaller version) and color palette variants (grayscale, black&white) got removed to reduce the pack's size and generation time with Color Mapper.

Special thanks to Steve Wright, who commissioned the pack without exclusivity, allowing me to release the assets for others to use, too! This documentation will go into more detail on how to find the files you want and how to change and re-export them to fit your needs.

Content Structure

The asset pack contains a folder called Sources and in it, you'll find the three folders input, output and misc. While misc contains a list of icon names ordered by their ID, the input and output folders are the ones relevant for a Color Mapper setup. If you don't use Color Mapper, you can head straight to the output-folder and use the files there, but if you do, please refer to the section Re-exporting the Icons with Color Mapper further down.

In the output folder, you'll find the icons separated by their type, which is a combination of their shape (circle, diamond, square), their border (thick, thin) and if they have a name band or not: these folders are named terrain_icons.standard_(circle/diamond/square)_(thick/thin) with an additional _named in case they are named. As an example, the folder terrain_icons.standard_diamond_thick_named contains the terrain icons in the diamond shape, with a thick border and a name band on top of them. The mini icons are separated by shape and are available in the folders named terrain_mini_icons.standard_(circle/diamond/square).

After we had a look at the output-folder, let's check the content structure of the input-folder. At input\terrain_icons you'll find all the Krita (.kra) and Photoshop (.psd) files as well as image file exports (.png) of all the single layers of every icon. Likewise, the folder input\terrain_mini_icons contains these files for the mini icons. These files are required if you want to use Color Mapper. The Color Mapper Files standard_circle_thick_named.colormapper.txt (etc.) tell Color Mapper how to assemble the icons.

Re-exporting the Icons with Color Mapper

If you already used Color Mapper before, please get the update before using it with the Terrain Icons pack, as the update contains the new features Masking and Absolute Layers required by this asset pack.

If you want to use the icons the way they are, refer to the Sources\output-folder. But if you want to change the icons, for example replace some or all of the colors, you can use my batch image editing tool Color Mapper. It can also re-size, re-scale and re-position icons, all in one go.

Color Mapper assembling an icon from its single parts.

Color Mapper assembling an icon, but also switching out some of the colors.

To make Color Mapper work, the Isle of Lore 2: Terrain Icons download includes the folders Sources\input\terrain_icons and Sources\input\terrain_mini_icons, both containing the Krita (.kra) and Photoshop (.psd) source files as well as png exports of the single layers and Color Mapper Files (.colormapper.txt) containing information on how Color Mapper is going to assemble the files.

Color Mapper isn't a part of this asset pack and instead has to be downloaded here, which allows me to release updates for Color Mapper independently of the projects it is used in. The download includes a Documentation-folder with exact instructions on how to use Color Mapper, but the workflow in general is at follows:

  1. Setup Color Mapper's configuration file so it knows where to find the Sources\input and Sources\output folders.
  2. Running Color Mapper, which will gather all the *.colormapper.txt-files in the input folder and assemble the png files from the single layer files with colors referenced in the *.colormapper.txt-file and defined in Color Mapper's configuration file. The assembled png files will be put in the output-folder.
  3. After checking the results, the files from the output-folder are ready to use.

If you want to make changes to the icons themselves, you can open the Krita or Photoshop files, edit them and export the single layers accordingly. Make sure to keep the naming scheme terrain_icon_(name)_(id)_(layer).png intact, for example terrain_icon_pine_forest_0_line.png for the line-layer of the pine forest icon which has the id 0. The mini icons are named terrain_mini_icon_(name)_(id)_(layer).png accordingly.

Both the icons and the mini icons use Color Mapper's mask feature to cut out the circle and diamond shapes of the icons, as well as absolute layers to reference the borders and name bands, except the name itself, which exists as a normal layer for every icon.
