Introduction

The asset pack Isle of Lore 2: Strategy Figures is a set of 125 figures compatible with my hex tile set Isle of Lore 2: Hex Tiles. The figures are available as png files as well as both Krita (.kra) and Photoshop (.psd) source files. Every figure comes in 3 variants: without a white outline (or border), with a fully encompassing outline and an outline which leaves out the bottom part of the figure. They also come in an additional version with extra padding, so they can be easily overlayed on top of the Isle of Lore 2 Hex Tiles. The asset pack is compatible with my batch image editing tool Color Mapper, so that you can batch edit the figures and change their color palette with minimal effort; the required Color Mapper Files are included!

If you bought the pack when it came out, you may notice some differences. First and foremost, the color palette is a bit darker and more saturated, making it compatible with the Isle of Lore 2: Hex Tiles update. The figures were completely re-drawn in a higher resolution (250% the size), too! Some semi-official figures like dwarves and halflings are now included and some further figures were added, like the catapult, trebuchet and ballista.

Special thanks to all the people providing ideas for figures as well as Dave Mauro and Conscir, who commissioned additional figures with non-exclusivity, allowing me to add these figures to the pack for everyone to use! Through Dave Mauro, the following figures got added: Villager Female, Princess, Queen, Boy, Girl, Harpy, Sleeping Dragon, Eyeball, Giant Worm, Gnoll, Dwarf, Elf and Halfling. Conscir's commission contributed the Beast Tamer, Jester and Ringmaster.

This documentation will go into more detail on how to find the files you want and how to change and re-export them to fit your needs.

Content Structure

The asset pack contains a folder called Sources and in it, you'll find the three folders input, output and misc. While misc contains a list of figure names ordered by their ID, the input and output folders are the ones relevant for a Color Mapper setup. If you don't use Color Mapper, you can head straight to the output-folder and use the files there, but if you do, please refer to the section Re-exporting the Figures with Color Mapper further down.

In the output folder, you'll find the figures separated by the type of outline: figures.standard_full (full outline), figures_standard_none (no outline) and figures.standard_semi (outline except at the bottom). If you want the figures as overlays for the Isle of Lore 2: Hex Tiles set, please refer to the folders starting with overlay_figures. In fact, if you use my hex tile set or Tile Builder, you may already have a folder called HexMapMaker somewhere. If so, you can move the folders named figures.standard_xxx to HexMapMaker\Elements\Isle of Lore 2 and the folders named overlay_figures.standard_xxx to HexMapMaker\Tiles\Isle of Lore 2.

The figures.standard_xxx-folders contain a meta file meta.txt providing the distance of the figures from the bottom of the image file, which is useful if you want to render them in a game engine with proper sorting.

After we had a look at the output-folder, let's check the content structure in the input-folder. At input\figures you'll find all the Krita (.kra) and Photoshop (.psd) files as well as image file exports (.png) of all the single layers of every figure. They are required if you want to use Color Mapper. The Color Mapper Files standard_full.colormapper.txt (etc.) tell Color Mapper how to assemble the figures. Special composite files ending on .composite.colormapper.txt represent groups of figures, like a group of people, and they tell Color Mapper how to assemble these group figures.

The folder input\overlay_figures contains the Color Mapper Files to let Color Mapper create the tile overlay variants of the figures.

Re-exporting the Figures with Color Mapper

If you already used Color Mapper before, please get the update before using it with the Strategy Figures pack. First, because there are two new colors which have to be added to Color Mapper's configuration file, namely iol2_skin and iol2_smoke, secondly because the pack uses a new feature for creating figure groups, which the previous version of Color Mapper didn't have.

If you want to use the figures the way they are, refer to the Sources\output-folder. But if you want to change the figures, for example replace some or all of the colors, you can use my batch image editing tool Color Mapper. It can also re-size, re-scale and re-position figures and even create figure compositions, like a group of people, all in one go.

Color Mapper assembling a villager from its single parts.

Color Mapper assembling a villager, but also switching out the color of one of its parts.

To make Color Mapper work, the Isle of Lore 2: Strategy Figures download includes a folder Sources\input\figures containing the Krita (.kra) and Photoshop (.psd) source files as well as png exports of the single layers, Color Mapper Files (.colormapper.txt) containing information on how Color Mapper is going to assemble the files and Composite Files (.composite.colormapper.txt) telling Color Mapper how to assemble composites like a group of people from single figures.

Color Mapper isn't a part of this asset pack and instead has to be downloaded here, which allows me to release updates for Color Mapper independently of the projects it is used in. The download includes a Documentation-folder with exact instructions on how to use Color Mapper, but the workflow in general is at follows:

  1. Setup Color Mapper's configuration file so it knows where to find the Sources\input and Sources\output folders.
  2. Running Color Mapper, which will gather all the *.colormapper.txt-files in the input folder and assemble the png files (and composites) from the single layer files with colors referenced in the *.colormapper.txt-file and defined in Color Mapper's configuration file. The assembled png files will be put in the output-folder.
  3. After checking the results, the files from the output-folder can be used or copied over to their corresponding HexMapMaker-folder (see section Content Structure).

Make sure to keep the naming scheme figure_(name)_(id)_(layer).png intact, for example figure_villager_0_line.png for the line-layer of the villager figure which has the id 0.
